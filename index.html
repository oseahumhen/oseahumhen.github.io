Software Engineer, Cambridge, United Kingdom

I am Software Engineer with three years working experience. Currently, I work with a team involved in the design, deployment, support and maintenance of a Python-based test automation software product used for testing firmware running on Bluetooth chips. In my current role I do the following:

  • Actively involved in the design and Implementation of a Python based test automation system. This Involves new feature development, discovering and fixing bugs.
  • Develop embedded APIs needed by Test Engineers to carry out Bluetooth firmware testing. These APIs wrap around low-level firmware APIs.
  • Develop test automation APIs for android phones using Appium, Selenium, Android Debug Bridge (ADB) in Python.
  • Design and setup hardware infrastructure needed to run test automation software. This involves ensuring devices under test can be totally controlled remotely. This is done by making design decision regarding hardware needed for remote power cycling, factory resetting e.g programmable relays and USB hubs. This also involves developing APIs and scripts needed to programmatically control the chosen hardware.
  • Manage Continuous Integration jobs using Jenkins.
  • Support test engineers in their use of the automated test system. This involves assisting in machine setup, writing new feature documentation and troubleshooting technical issues that arise during test system usage.
  • Create/maintain docker images used to deploy test automation system.

I have a keen interest in understanding things from first principle. I am constantly looking forward to working with cutting edge technologies and expanding knowledge base. I have basic knowledge of machine learning concepts. Currently, I am developing skills in the area of cloud computing.
